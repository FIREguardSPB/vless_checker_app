# VLESS Checker для Android

Минимальное Android-приложение на Kotlin.

Теперь умеет:
- брать список VLESS-ссылок из `assets/servers.txt` или из импортированного файла;
- показывать и редактировать список прямо в приложении;
- проверять только первый рабочий вариант и сразу копировать его в буфер;
- выполнять полную проверку всех ссылок и показывать все рабочие;
- по опции удалять нерабочие ссылки из списка после полной проверки;
- запускать автопроверку по таймеру через WorkManager;
- показывать системное уведомление при нахождении рабочей ссылки.

Как именно проверяется доступность:
- `security=none` -> TCP connect к `host:port`;
- `security=tls` -> TCP connect + TLS handshake с `sni`, если он есть;
- `security=reality` -> только TCP connect.

Важно:
- это **не полный VLESS/Xray клиент**;
- для `reality` и некоторых нестандартных конфигов проверка приблизительная;
- для 100% проверки именно успешного VLESS-сеанса лучше интегрировать `xray-core` или `sing-box`.

## Как открыть проект
1. Открой папку `vless_checker_app` в Android Studio.
2. Дождись Gradle Sync.
3. Запусти на устройстве или эмуляторе.

## Где лежит список по умолчанию
`app/src/main/assets/servers.txt`

## Что добавлено в версии 1.1
- автопроверка по таймеру;
- импорт списка из файла на телефоне;
- полная проверка всех рабочих ссылок;
- опция удаления нерабочих ссылок после полного сканирования.


## Как получить APK без ручной настройки

### Вариант 1 — через GitHub Actions
1. Загрузи проект в GitHub-репозиторий.
2. Открой вкладку **Actions**.
3. Запусти workflow **Build Android Debug APK**.
4. После завершения скачай artifact `vless-checker-debug-apk`.

### Вариант 2 — локально
1. Установи Android Studio.
2. Убедись, что установлены:
   - Android SDK Platform 34
   - Android SDK Build-Tools 34.0.0
   - Java 17
3. В корне проекта запусти:
   ```bash
   ./build_debug_locally.sh
   ```
4. APK появится в `app/build/outputs/apk/debug/`.

## Почему в архиве нет готового APK
В этом окружении нет Android SDK/Gradle и нет доступа в сеть для их загрузки, поэтому готовый APK здесь не собирается. Но проект подготовлен так, чтобы собрать debug APK автоматически на GitHub или локально без правок кода.
