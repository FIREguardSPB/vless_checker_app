# VLESS Checker для Android

Минимальное Android-приложение на Kotlin.

Теперь умеет:
- брать локальный список из `assets/servers.txt` или импортированного файла;
- автоматически подтягивать удалённые списки:
  - `available(top100)`
  - `white-list_available(top100)`
- выбирать источник списка прямо в интерфейсе;
- проверять все ссылки и копировать в буфер **самую быструю рабочую**;
- выполнять полную проверку всех ссылок и показывать для каждой статус и время отклика;
- по опции убирать нерабочие строки из текущего списка на экране после полной проверки;
- запускать автопроверку по таймеру через WorkManager для выбранного источника;
- показывать системное уведомление при нахождении самой быстрой рабочей ссылки;
- передавать найденную самую быструю ссылку в совместимый VPN-клиент через `ACTION_VIEW`, а если прямого обработчика нет — через системный share chooser.

Как именно проверяется доступность:
- `security=none` -> TCP connect к `host:port`;
- `security=tls` -> TCP connect + TLS handshake с `sni`, если он есть;
- `security=reality` -> только TCP connect.

Поддержка форматов в текущей версии:
- `vless://`
- `trojan://`
- `vmess://` (разбор base64 JSON для host/port/tls)

Важно:
- это **не полный VLESS/Xray клиент**;
- для `reality` и некоторых нестандартных конфигов проверка приблизительная;
- время отклика — это время подключения/TLS handshake, а не полноценная скорость канала;
- прямое создание системного VLESS-профиля Android не реализовано;
- для полноценного VPN-подключения внутри самого приложения нужна отдельная реализация на `VpnService` + встроенный `xray-core`/`sing-box`.

## Как открыть проект
1. Открой папку `vless_checker_app_buildready` в Android Studio.
2. Дождись Gradle Sync.
3. Запусти на устройстве или эмуляторе.

## Где лежит список по умолчанию
`app/src/main/assets/servers.txt`

## Что добавлено в версии 1.3
- выбор источника списка: локальный / `available(top100)` / `white-list_available(top100)`;
- автоматическое подтягивание удалённых списков перед проверкой;
- кэширование последнего удалённого списка, чтобы можно было использовать его при временной ошибке загрузки;
- кнопка передачи fastest-ссылки в совместимый VPN-клиент;
- кнопка открытия системных настроек VPN;

## Как получить APK без ручной настройки

### Вариант 1 — через GitHub Actions
1. Загрузи проект в GitHub-репозиторий.
2. Открой вкладку **Actions**.
3. Запусти workflow **Build Android Debug APK**.
4. После завершения скачай artifact `vless-checker-debug-apk`.

### Вариант 2 — локально
1. Установи Android Studio.
2. Убедись, что установлены:
   - Android SDK Platform 34
   - Android SDK Build-Tools 34.0.0
   - Java 17
3. В корне проекта запусти:
   ```bash
   ./build_debug_locally.sh
   ```
4. APK появится в `app/build/outputs/apk/debug/`.

## Почему в архиве нет готового APK
В этом окружении нет Android SDK/Gradle и нет доступа в сеть для их загрузки, поэтому готовый APK здесь не собирается. Но проект подготовлен так, чтобы собрать debug APK автоматически на GitHub или локально без правок кода.


## Что изменено в версии 1.4

- удалённые списки переключены на `available(top100)` и `white-list_available(top100)`;
- после полной проверки каждый конфиг показывается отдельной карточкой;
- короткое нажатие по карточке копирует одну чистую ссылку в буфер для вставки в v2RayTun;
- долгое нажатие пытается открыть конфиг в v2RayTun или другом совместимом клиенте.

- добавлены кнопки управления v2RayTun через deeplink-команды start/stop/restart.
